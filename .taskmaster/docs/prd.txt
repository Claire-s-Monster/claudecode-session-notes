# Overview
The session-notes MCP server is a specialized Model Context Protocol (MCP) server designed to facilitate comprehensive collection and analysis of ClaudeCode session workbooks. This server addresses the critical need for understanding AI agent behavior, identifying missing tools and capabilities, and optimizing agent communication patterns during development sessions.

The primary problem this solves is the lack of visibility into how AI agents interact, what tools they need but don't have access to, and how development sessions progress. This information is crucial for improving agent performance, identifying gaps in tooling, and understanding workflow patterns for better AI-assisted development.

Target users include:
- AI development framework maintainers
- ClaudeCode users seeking to optimize their workflows
- Researchers studying AI agent behavior patterns
- Development teams analyzing AI-assisted coding efficiency

# Core Features

## Session Workbook Collection
- Automatically captures session metadata, tool usage, and agent interactions
- Stores structured session data in organized file hierarchies
- Tracks session duration, tool invocation patterns, and outcome metrics
- Enables retrospective analysis of development sessions

## Agent Logging Framework
- Individual agent execution tracking with unique agent IDs
- Captures agent communication patterns and decision trees
- Logs tool requests, successes, failures, and missing tool scenarios
- Records prime context usage and tuning requirements

## Missing Tool Detection
- Identifies tools that agents attempt to use but aren't available
- Catalogs script requests that couldn't be fulfilled
- Tracks MCP server capabilities that would improve workflow
- Generates recommendations for tooling improvements

## Analytics and Reporting
- Session pattern analysis and workflow optimization insights
- Agent performance metrics and communication efficiency reports
- Tool usage statistics and missing capability analysis
- Development velocity and quality correlation studies

# Technical Architecture

## System Components

### MCP Server Core
- FastMCP 2.0-based server implementation with Python
- Decorator-based development pattern for 60-70% faster implementation
- RESTful API endpoints for session data management
- WebSocket support for real-time session monitoring
- Plugin architecture for extensible data collection
- State-of-the-art MCP server framework with modern Python patterns

### Data Collection Engine
- Session metadata collector (timestamps, user, environment)
- Agent interaction tracker (communication patterns, decision flows)
- Tool usage monitor (invocations, parameters, success/failure rates)
- Missing capability detector (failed tool requests, script needs)

### Storage Layer
- File-based storage system with hierarchical organization
- JSON-based data format for easy parsing and analysis
- Structured directory layout: `.claude/session-notes/<session-id>/`
- Agent-specific logging: `.claude/session-notes/<session-id>/agents/<agent-id>/`

### Analytics Engine
- Pattern recognition algorithms for workflow analysis
- Statistical reporting for agent performance metrics
- Trend analysis for tool usage and missing capabilities
- Recommendation engine for workflow optimization

## APIs and Integrations

### MCP Protocol Endpoints
- `notes://sessions/list` - List all captured sessions
- `notes://sessions/get/{id}` - Retrieve specific session data
- `notes://agents/list/{sessionId}` - List agents for a session
- `notes://agents/get/{agentId}` - Retrieve agent execution logs
- `notes://analytics/report` - Generate analytics reports
- `notes://tools/missing` - Get missing tool recommendations

### Integration Points
- ClaudeCode session hooks for automatic data collection
- Git integration for correlating code changes with session data
- CI/CD pipeline integration for quality correlation analysis
- Future database migration endpoints for scalability

**Risk: Performance Impact on Active Sessions**
- Mitigation: Asynchronous data collection with minimal blocking operations
- Mitigation: Configurable logging levels to balance detail vs. performance
- Mitigation: Background data processing to avoid session interruption

**Risk: Data Storage Growth**
- Mitigation: Configurable retention policies with automatic cleanup
- Mitigation: Data compression for older sessions
- Mitigation: Optional cloud storage integration for long-term archival

**Risk: MCP Protocol Compatibility**
- Mitigation: Comprehensive testing with different ClaudeCode versions

- Mitigation: Fallback mechanisms for unsupported protocol features

## Technical Specifications

### File Structure Design
```
.claude/session-notes/
├── sessions.index.json          # Session registry and metadata
├── <session-id>/
│   ├── session.json            # Session metadata and summary
│   ├── tools.json              # Tool usage log
│   ├── agents/
│   │   ├── <agent-id>/
│   │   │   ├── execution.json  # Agent execution log
│   │   │   ├── interactions.json # Agent communication log
│   │   │   └── context.json    # Prime context usage
│   │   └── communications.json # Inter-agent communications
│   └── analytics/
│       ├── patterns.json       # Identified workflow patterns
│       └── recommendations.json # Generated recommendations
└── reports/
    ├── daily/                  # Daily analytics reports
    ├── weekly/                 # Weekly trend analysis
    └── custom/                 # User-generated reports

# Note: This file structure is managed by Python-based FastMCP 2.0 server
# with PIXI environment for dependency and runtime management
```

### Performance Requirements
- Session data collection latency: <10ms per operation
- Query response time: <100ms for basic queries, <1s for analytics
- Storage efficiency: <1MB per average development session
- Memory usage: <50MB baseline, <100MB during active analytics processing

### Security Considerations
- No sensitive code content stored, only metadata and patterns
- Session isolation through directory-based access control
- Optional encryption for sensitive development environments
- Audit logging for all data access and modification operations
